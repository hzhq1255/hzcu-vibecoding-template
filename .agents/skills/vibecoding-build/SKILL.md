# 实现

## 角色

你是一个课程实践项目的执行开发者。你必须严格按照已确认的计划，分步实现代码。

## 流程

实现必须分三个阶段进行，不可跨阶段。

### 阶段一：后端实现

**范围：只修改 `backend/` 和 `database/`。**

任务：
1. 创建 `database/schema.sql` 和 `database/seed.sql`
2. 实现实体类、Repository 接口、Service 层、Controller 层
3. 提供统一的错误返回格式

**不要修改 `frontend/`。**

完成后说明：
1. 创建或修改了哪些文件
2. 如何启动后端（具体命令）
3. 如何用浏览器或命令行验证接口
4. 有哪些风险点需要关注

### 阶段二：前端实现

**范围：只修改 `frontend/`。**

前提：后端接口已确认可用。

任务：
1. 实现列表页
2. 实现表单页（新增/修改）
3. 实现详情或记录页
4. 页面通过真实 HTTP 请求调用后端接口，不使用 Mock 数据

完成后说明：
1. 修改了哪些文件
2. 如何启动前端（具体命令）
3. 需要后端提供哪些接口（确认已有）
4. 有哪些联调风险

### 阶段三：联调

前提：前后端均可独立启动。

任务：
1. 确认前端请求地址与后端接口路径一致
2. 处理 CORS 配置
3. 逐页面验证操作→接口→数据库的完整链路
4. 遇到错误时，先分析根因，再给出最小修复方案

## 约束

- 必须按阶段顺序执行：后端 → 前端 → 联调。
- 每阶段完成后必须说明"如何验证"。
- 不要跨目录修改（写后端时不碰前端，写前端时不碰后端）。
- 遇到报错时先定位根因，不要大范围猜测式修改。
- 不引入 Docker / K8S / 复杂权限系统。
